# Database Normalization & Schema Implementation

### Overview
This project focuses on the design and implementation of a relational database schema. The objective was to achieve Third Normal Form (3NF) to minimize data redundancy, ensure referential integrity, and support efficient business logic for customer orders and supplier relationships.

### Implementation Details
* **Target Engine:** SQL Server / Standard SQL
* **Key Concepts:** 1NF, 2NF, 3NF normalization, foreign key constraints, and transactional consistency.

### Development & Research
The architecture of this schema was developed by applying foundational database principles. The following resources were used to guide the normalization process and verify schema constraints:

#### Foundational Literature
* Elmasri, Ramez, and Shamkant B. Navathe. *Fundamentals of Database Systems*. 7th ed., Pearson, 2016.
* Gibbs, Martin. "Third Normal Form in DBMS with Examples." *Study.com*.
* Odugbesan, Temitayo. "What is Normal Form in DBMS? - Types & Examples." *Study.com*.

#### Development Methodology
During the design and implementation phase, I utilized AI tools as a development assistant to refine normalization strategies, verify ERD logic, and ensure SQL syntax accuracy. I maintained full oversight of the development process, reviewing and validating all outputs against database management principles to ensure the final schema met professional standards.

### How to Run
1. Open your preferred SQL interface (e.g., SQL Server Management Studio).
2. Execute the `company_3nf_schema.sql` script to generate the tables.
3. Run `normalization_and_schema_implementation.sql` to execute the data population and test the relational constraints.
